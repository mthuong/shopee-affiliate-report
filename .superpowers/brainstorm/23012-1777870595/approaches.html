Architecture Approaches

Choose the approach that fits your needs best

A

⭐ Recommended: Lightweight — No Image Storage

Upload images → send to Gemini API (via Next.js server action) for parsing → show extracted orders → save only order data to Supabase. Images are never stored permanently.

  • ✅ Zero storage costs
  • ✅ Simplest architecture
  • ✅ Fast to build
  • ⚠️ Can't review original screenshots later
B

Full — Image Storage + Parsing

Upload images to Supabase Storage → Gemini parses them → extracted orders saved to Postgres. Original screenshots permanently accessible.

  • ✅ Keep original proof screenshots
  • ✅ Re-parse images if needed
  • ⚠️ Supabase Storage costs (small but non-zero)
  • ⚠️ More complex setup
C

Manual-first — Parsing as Enhancement

Primarily a manual entry app. Optionally upload images for auto-fill. Focus is on the order management, client tracking, and Excel export features.

  • ✅ No AI API dependency
  • ✅ Works offline-ish
  • ⚠️ Manual entry is slow for many orders
  • ⚠️ Defeats the purpose of image upload
